# Quoridor AI Agent 訓練指南

本項目使用 OpenAI Gym 和 Stable Baselines3 (SB3) 來訓練一個能夠玩 Quoridor 遊戲的 AI Agent。

## 環境設置

### 1. 安裝依賴

```bash
pip install stable-baselines3 gymnasium sb3-contrib
```

### 2. 項目結構

```
# 目前的結構
```

## 快速開始

### 訓練模型

```bash
# 使用默認參數訓練（100,000 時間步）
python train.py --mode train

# 自定義訓練時間步數（例如 500,000 步）
python train.py --mode train --timesteps 500000
```

**訓練配置說明：**
- **演算法：** PPO (Proximal Policy Optimization)
  - 優點：穩定、適合離散動作空間、易於調優
  - 推薦用於遊戲 AI 訓練

- **網路結構：** MlpPolicy（全連接網路）
  - 適合棋盤狀態表示
  - 可減少計算開銷

- **超參數：**
  - 學習率：3e-4
  - 批次大小：64
  - 每次更新步數：2048
  - 折扣因子 (gamma)：0.99
  - GAE 參數 (lambda)：0.95

### 評估模型

```bash
# 評估訓練好的模型（10 局遊戲）
python train.py --mode evaluate

# 評估並渲染遊戲過程
python train.py --mode evaluate --num-episodes 20 --render

# 評估自定義模型路徑
python train.py --mode evaluate --model-path models/quoridor_ppo_final.zip --num-episodes 10
```

### 與 AI 對戰

```bash
# 與 AI 進行一局遊戲（渲染棋盤）
python train.py --mode play
```

## 文件說明

### quoridor_env.py

**QuoridorEnv 類別特性：**

1. **觀察空間 (Observation Space)**
   - 類型：Box(168,)
   - 包含：
     - 玩家1位置 (2)
     - 玩家2位置 (2)
     - 玩家1剩餘牆體 (1)
     - 玩家2剩餘牆體 (1)
     - 棋盤狀態矩陣 (81)：玩家位置和牆體信息
     - 可走位置遮罩 (81)：合法移動位置

2. **動作空間 (Action Space)**
   - 類型：Discrete(209)
   - 0-80：移動動作（9×9 格子）
   - 81-144：放置橫牆（8×8 位置）
   - 145-208：放置直牆（8×8 位置）

3. **獎勵函數 (Reward Function)**
   - 基於 `board.evaluate_action_reward()`
   - 考慮自身進度和對手阻礙
   - 獲勝時獲得 +100 獎勵
   - 非法動作獲得 -1 懲罰

4. **合法動作約束**
   - 使用 `board.get_legal_actions_mask()` 獲取合法動作
   - 非法動作會導致立即遊戲結束並獲得負獎勵

### train.py

**訓練腳本功能：**

1. **train_ppo_agent()**
   - 主要訓練函數
   - 支持自定義超參數
   - 定期保存檢查點
   - TensorBoard 監控支持

2. **evaluate_agent()**
   - 評估模型性能
   - 統計勝率和平均獎勵
   - 可選渲染遊戲過程

3. **play_game_with_agent()**
   - 自由遊戲模式
   - 實時渲染棋盤狀態
   - 用於調試和觀察 AI 行為

## 常見問題

### 1. 環境導入錯誤
```python
# 確保 rule.py 在同一目錄中
ImportError: cannot import name 'Board'
```

**解決方案：** 檢查 rule.py 和 quoridor_env.py 在同一目錄

### 2. 訓練很慢
- 默認使用 CPU，預計 30-60 分鐘訓練 100,000 步
- 可使用 GPU 加速（需要 CUDA 支持）

### 3. 模型性能不好
- 增加訓練時間步數
- 調整學習率：嘗試 1e-4 到 1e-3
- 調整批次大小或 n_steps

### 4. 記憶體不足
- 減少 n_steps 參數（預設 2048）
- 嘗試使用 batch_size=32 或更小
