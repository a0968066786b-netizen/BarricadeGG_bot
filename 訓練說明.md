# Quoridor AI Agent 訓練指南

本項目使用 Gymnasium 和 Stable Baselines3 (SB3) 來訓練一個能夠玩 Quoridor 遊戲的 AI Agent。

## 項目結構

```
BarricadeGG_bot/
├── setup.py                    # 用於安裝 Package
├── requirements.txt            # Python 依賴
├── quoridor_core/              # 核心邏輯 Package
│   ├── __init__.py             # 包初始化，導出主要類和函數
│   ├── env.py                  # Gymnasium 環境實現（QuoridorEnv）
│   └── rules.py                # 遊戲規則實現（Board, Player, Wall 等）
├── scripts/                    # 執行腳本
│   ├── __init__.py
│   ├── train.py                # 訓練腳本
│   └── evaluate.py             # 評估和遊玩腳本
├── models/                     # 存放訓練好的模型 (.zip)
├── logs/                       # 存放 TensorBoard 日誌
├── 遊戲規則.md                 # 遊戲規則文檔
└── 訓練說明.md                 # 本文件
```

## 環境設置

### 1. 安裝依賴

```bash
# 安裝所有依賴
pip install -r requirements.txt

# 或使用 setup.py 安裝（包括開發模式）
pip install -e .
```

### 2. 驗證安裝

```bash
python -c "from quoridor_core import QuoridorEnv; print('安裝成功！')"
```

## 快速開始

### 1. 訓練模型

```bash
# 進入 scripts 目錄（或直接在根目錄運行）

# 使用默認參數訓練（100,000 時間步）
python scripts/train.py

# 自定義訓練時間步數（例如 500,000 步）
python scripts/train.py --timesteps 500000

# 自定義其他超參數
python scripts/train.py --timesteps 100000 --learning-rate 3e-4 --batch-size 64 --n-steps 2048
```

**訓練配置說明：**
- **演算法：** PPO (Proximal Policy Optimization)
  - 優點：穩定、適合離散動作空間、易於調優
  - 推薦用於遊戲 AI 訓練

- **網路結構：** MlpPolicy（全連接網路）
  - 適合棋盤狀態表示
  - 可減少計算開銷

- **超參數：**
  - 學習率：3e-4
  - 批次大小：64
  - 每次更新步數：2048
  - 折扣因子 (gamma)：0.99
  - GAE 參數 (lambda)：0.95

### 2. 評估模型

```bash
# 評估訓練好的模型（10 局遊戲）
python scripts/evaluate.py

# 評估自定義數量的遊戲（例如 20 局）
python scripts/evaluate.py --num-episodes 20

# 評估並顯示遊戲過程
python scripts/evaluate.py --num-episodes 10 --render

# 評估自定義模型路徑
python scripts/evaluate.py --model-path models/custom_model.zip --num-episodes 10
```

### 3. 與 AI 對戰

```bash
# 與訓練好的 AI 進行一局遊戲（會顯示棋盤）
python scripts/evaluate.py --mode play

# 與自定義模型對戰
python scripts/evaluate.py --mode play --model-path models/custom_model.zip
```

## 文件說明

### quoridor_core/rules.py
遊戲核心邏輯實現：
- **Board**：棋盤類別，管理遊戲狀態、可走位置、獎勵計算
- **Player**：玩家類別，記錄位置、剩餘牆體、目標行
- **Wall**：牆體類別，表示棋盤上的牆體
- **函數**：座標轉換、動作 ID 轉換、合法性判斷、最短路徑計算

### quoridor_core/env.py
Gymnasium 環境實現（QuoridorEnv）：

**觀察空間 (Observation Space)**
- 類型：Box(168,)
- 包含：
  - 玩家1位置 (2)
  - 玩家2位置 (2)
  - 玩家1剩餘牆體 (1)
  - 玩家2剩餘牆體 (1)
  - 棋盤狀態矩陣 (81)：玩家位置和牆體信息
  - 可走位置遮罩 (81)：合法移動位置

**動作空間 (Action Space)**
- 類型：Discrete(209)
- 0-80：移動動作（9×9 格子）
- 81-144：放置橫牆（8×8 位置）
- 145-208：放置直牆（8×8 位置）

**獎勵函數**
- 基於 `board.evaluate_action_reward()`
- 考慮自身進度和對手阻礙
- 獲勝時獲得 +100 獎勵
- 非法動作獲得 -1 懲罰

### scripts/train.py
簡化的訓練腳本：
- 支持自定義超參數：時間步數、學習率、批次大小、步數
- 定期保存檢查點（每 10,000 步保存一次）
- TensorBoard 日誌支持（logs 目錄）
- 命令行參數配置

### scripts/evaluate.py
評估和遊玩腳本：
- **評估模式**：計算勝率、平均獎勵、平均步數
- **遊玩模式**：與訓練好的 AI 進行互動遊戲
- 支持渲染棋盤狀態
- 支持自定義模型路徑

## 在自己的代碼中使用 quoridor_core

### 作為 Python 包導入

```python
from quoridor_core import QuoridorEnv, Board, action_id_to_action

# 創建環境
env = QuoridorEnv()

# 重置環境
obs, info = env.reset()

# 執行動作
obs, reward, terminated, truncated, info = env.step(action)

# 訪問棋盤
board = env.board
legal_actions = board.get_legal_actions_mask()
```

### 自定義訓練循環

```python
from quoridor_core import QuoridorEnv
from stable_baselines3 import PPO

# 創建環境
env = QuoridorEnv()

# 訓練
model = PPO('MlpPolicy', env, learning_rate=3e-4)
model.learn(total_timesteps=100000)

# 保存
model.save('my_model')

# 加載並評估
model = PPO.load('my_model')
obs, _ = env.reset()
action, _ = model.predict(obs)
```

## 常見問題

### 1. 導入錯誤：找不到 quoridor_core

**症狀：** `ImportError: No module named 'quoridor_core'`

**解決方案：**
```bash
# 確保在項目根目錄運行腳本
# 或使用 setup.py 安裝
pip install -e .
```

### 2. 文件找不到：models/quoridor_ppo_final.zip

**症狀：** `FileNotFoundError: models/quoridor_ppo_final.zip`

**解決方案：** 先訓練模型，模型會保存在 models 目錄
```bash
python scripts/train.py --timesteps 100000
```

### 3. 訓練速度很慢

**原因：** 默認使用 CPU 訓練

**解決方案：**
- 使用 GPU（如有 CUDA）：修改 scripts/train.py，將 `device='cpu'` 改為 `device='cuda'`
- 減少訓練時間步數進行快速測試
- 使用更小的批次大小和 n_steps

### 4. 記憶體不足

**解決方案：**
```bash
# 減少批次大小和 n_steps
python scripts/train.py --batch-size 32 --n-steps 1024
```

### 5. 環境或模型不兼容

**症狀：** Gymnasium 或 Stable Baselines3 版本不匹配

**解決方案：**
```bash
# 重新安裝所有依賴
pip install --upgrade -r requirements.txt
```
