# 規則定義：

## 格子定義：
棋盤為9*9格子大小範圍的棋盤，其中每個球體所待的方格子使用代數記譜法
與西洋棋類似，以紅色球玩家視角看待，a~i為column的所處代碼(順序由左至右)、1~9為row的所處代碼(順序由下到上)，先表示column再表示row，例如a1、b2、c3...以此類推

## 牆面規則定義：
每個牆體為3位數代碼，首位只會為h或v(h代表橫向、v代表直向)，
第二與第三位則是代表牆體交叉點所在的代碼位置，也一樣按照代數記譜法，第二位以紅色玩家視角看待column所處代碼 a~h(順序由左至右，且牆a在格子a與格子b之間、牆b在格子b與格子c之間...)，第三位則是row的所處代碼1~ 8(順序由下到上，且牆1在格子1與格子2之間、牆2在格子2與格子3之間...)

## 順序定義：
紅色球玩家為player1、藍色球玩家為player2，
由player1開始操控，接著換到player2操控，然後在回歸player1操控，以此類推，直到遊戲結束

當該玩家操控時可以選擇移動一格(參考移動規則)或者是放置一個牆體(參考放置牆體規則)


## 移動規則 (Movement)：

球選擇移動的時候每次只能移動一格（方向為上下左右）。

「已放置的牆體」會阻擋球移動，球無法穿過牆體。
雙方的上下左右四個方向若有對手球體，且雙方球體之間若沒有牆體阻擋，則可直接穿過對手球體移動過去(例如在b2位置的紅球要移動，原本只能移動到a2、c2、b1、b3，但是b3有藍球的話(也就是在紅球上方)，則可以移動到a2、c2、b1、b4(穿過對手球體移動到b4))但如果阻攔的球的後方有牆體(也就是能穿過但是不能直線穿越)，則會改成斜角的方式穿越，例如(維持剛剛的例子，如果b4的籃球上方有牆體的話(牆體代碼為ha3或hb3)，則改成能移動到a2、c2、b1、a3、c3(穿過對手球體移動到b4被牆體阻擋了，所以原本穿到b4的格子變成只能穿到a3跟c3，但如果球穿過到a3或c3的格子被牆體阻攔，也就是籃球的左或右方也有牆體，導致斜角穿越也受阻礙則無法斜角該方向，那a3或c3則也可能無法穿越))。

## 放置牆體規則 (Placement)：

雙方玩家各自有10個牆體可以放置，輪到該玩家操控時可以選擇放置一個牆體，放置牆體的方式為指定牆體代碼(例如ha3、vb5...以此類推)，但是放置後該玩家可使用的牆體-1，且若該玩家可使用的牆體數量為0的時候，則該玩家就只能移動，無法再放置牆體了。

牆體會改變路徑成本，使其部分路徑無法通行或消耗更多步數才能到達終點。但是一旦該位置放置牆體後，其中一名玩家無法走到終點(也就是被封鎖了)，則該牆體放置無效，必須重新放置。

## 勝負條件 (Win Condition)：

紅色球玩家(player1)的終點為row 9的格子(紅球到達a9~i9任意一個即到達終點)，
藍色球玩家(player2)的終點為row 1的格子(藍球到達a1~i1任意一個即到達終點)，
當其中一名玩家的球體成功移動到自己的終點位置時，該玩家獲勝，遊戲結束。

## 針對AI的獎勵、懲罰定義與目標：

目標：讓AI學會在遊戲中獲勝(以最小路徑成本為目標)，並且盡可能地阻止對手獲勝(放置牆體來讓對手的最路徑成本增加)。

靠近終點(因為離終點更近，最小路徑成本更低)：+ 分數。

成功放置牆體阻礙對手(讓對手的最小路徑成本增加)：+ 分數。

被封鎖(自己的路徑被封鎖的最小路徑成本增加)：- 分數。

## 串接用的規則：

### 串接AI用的：
### 動作 ID 映射規範：
為了確保 AI 訓練與程式對接的一致性，所有動作必須對應到一個 0 ~ 208 的整數 ID：

ID 0 ~ 80：移動動作 (Move)
    對應棋盤格子 a1 ~ i9。
    計算方式：ID = row_index * 9 + col_index。
    例如：$a1$ 為 $0$，$b1$ 為 $1$，$i9$ 為 $80$。

ID 81 ~ 144：放置橫牆 (Horizontal Wall)
    對應牆體位置 $ha1 \sim hh8$。
    計算方式：$ID = 81 + (row\_index \times 8 + col\_index)$。
    例如：$ha1$ 為 $81$，$hh8$ 為 $144$。
ID 145 ~ 208：放置直牆 (Vertical Wall)
    對應牆體位置 $va1 \sim vh8$。
    計算方式：$ID = 145 + (row\_index \times 8 + col\_index)$。
    例如：$va1$ 為 $145$，$vh8$ 為 $208$。

### 合法動作遮罩：
為了讓外部系統（AI 訓練器、前端 GUI）能即時得知目前的合法操作
系統需提供一個長度為 209 的布林列表（Boolean List）：

格式：[bool, bool, ...] (共 209 個元素)。

映射關係：對應前述209個的 **動作 ID** 映射規範 的索引位置。

用途：

AI 訓練：模型僅能從數值為 True 的索引中選擇動作，避免無效探索。

前端 GUI：讀取此列表，將數值為 False 的格子或牆位設為「不可點擊」或「置灰」狀態。

### 狀態快照 (Board Snapshot)：
為了確保所有模組共享唯一的「真理來源」，系統需提供一個標準化的字典（Dict/JSON）結構：

player1_pos: 字符串（如 "e1"）。

player2_pos: 字符串（如 "e9"）。

walls_remaining: {"p1": int, "p2": int}。

placed_walls: {"h": [str], "v": [str]} (如 {"h": ["ha1", "hb2"], "v": []})。

current_turn: player1 或 player2。

winner: str（若無則為空字串）。
